# Plan: T-2026-134 — Create keyboard shortcut service for minigame controls

## Task

Multiple minigame specs reference keyboard shortcuts: Module Assembly (number keys 1-6 for slots, spacebar to grab), Wire Protocol (1-4 for wire types), Flow Commander (keyboard gate selection). Create a shared `KeyboardShortcutService` to register, manage, and display keyboard shortcuts, avoiding duplicated logic in each minigame.

## Architecture

**Approach:** A root-provided Angular service that listens to `keydown` events on the document and dispatches to registered callbacks. This mirrors the existing service pattern used by `HintService` and `DragDropService` -- stateful, signal-based, with an explicit `unregisterAll()` teardown method.

**Key decisions:**

1. **No `inject()` calls — instantiable with `new`** for test consistency with `HintService`, `DragDropService`, `ScoreCalculationService`. The service uses the global `document` directly and does NOT use `inject(DestroyRef)` or `inject(DOCUMENT)`. Since this is `providedIn: 'root'`, the listener lives for the app lifetime — `DestroyRef` cleanup is academic. A `destroy()` method is provided for test cleanup.
2. **Signal-based `isEnabled`** -- writable internally (`WritableSignal`), exposed as `Signal` readonly. When `false`, all registered shortcuts are ignored (use case: pause menu open).
3. **Map<string, ShortcutRegistration> for O(1) dispatch** -- key string (normalized to lowercase) maps to callback + label. Only one handler per key at a time (last registration wins), appropriate since only one minigame is active at a time.
4. **`getRegistered()` returns an array** (not a signal) for simplicity -- the help overlay will call it on demand.
5. **`unregisterAll()` resets `isEnabled` to `true`** -- ensures clean teardown for the next minigame.

**Why not RxJS `fromEvent`?** The project uses signals, not RxJS. A plain `addEventListener` with explicit cleanup is simpler and aligns with signal-first patterns.

## Detailed Implementation

### keyboard-shortcut.service.ts (~70 LOC)

```typescript
import { Injectable, signal, type Signal } from '@angular/core';

/** Describes a registered keyboard shortcut. */
export interface ShortcutRegistration {
  readonly key: string;    // normalized lowercase key (e.g., '1', ' ', 'arrowup')
  readonly label: string;  // human-readable label for help overlay
  readonly callback: () => void;
}

@Injectable({ providedIn: 'root' })
export class KeyboardShortcutService {
  private readonly _shortcuts = new Map<string, ShortcutRegistration>();
  private readonly _isEnabled = signal(true);
  readonly isEnabled: Signal<boolean> = this._isEnabled.asReadonly();

  private readonly _boundHandler = this._handleKeydown.bind(this);

  constructor() {
    document.addEventListener('keydown', this._boundHandler);
  }

  /** Register a keyboard shortcut. Last registration for a key wins. */
  register(key: string, label: string, callback: () => void): void {
    const normalizedKey = key.toLowerCase();
    this._shortcuts.set(normalizedKey, { key: normalizedKey, label, callback });
  }

  /** Clear all shortcuts and reset isEnabled to true. */
  unregisterAll(): void {
    this._shortcuts.clear();
    this._isEnabled.set(true);
  }

  /** Returns current registered shortcuts for help overlay display. */
  getRegistered(): readonly ShortcutRegistration[] {
    return [...this._shortcuts.values()];
  }

  /** Enable or disable all shortcut dispatch. */
  setEnabled(enabled: boolean): void {
    this._isEnabled.set(enabled);
  }

  /** Remove the keydown listener. Call in tests only (root service lives forever in prod). */
  destroy(): void {
    document.removeEventListener('keydown', this._boundHandler);
  }

  private _handleKeydown(event: KeyboardEvent): void {
    if (!this._isEnabled()) return;
    const registration = this._shortcuts.get(event.key.toLowerCase());
    if (registration) {
      event.preventDefault();
      registration.callback();
    }
  }
}
```

**No `inject()` calls** — the service is instantiable with `new KeyboardShortcutService()` in tests, matching `HintService` and `DragDropService` specs.

### keyboard-shortcut.service.spec.ts (~100 LOC)

**Test instantiation:** `new KeyboardShortcutService()` in `beforeEach`, `service.destroy()` in `afterEach` to remove the listener and prevent accumulation.

```typescript
import { vi } from 'vitest';
import { KeyboardShortcutService } from './keyboard-shortcut.service';

function dispatchKey(key: string): KeyboardEvent {
  const event = new KeyboardEvent('keydown', { key, bubbles: true, cancelable: true });
  document.dispatchEvent(event);
  return event;
}

describe('KeyboardShortcutService', () => {
  let service: KeyboardShortcutService;

  beforeEach(() => {
    service = new KeyboardShortcutService();
  });

  afterEach(() => {
    service.destroy();
  });

  // ... tests
});
```

**Test cases (10):**

1. **Creation** -- service is truthy, `isEnabled()` defaults to `true`.
2. **register + getRegistered** -- after registering 2 shortcuts, `getRegistered()` returns both with correct key/label.
3. **Dispatch** -- register a callback via `vi.fn()`, dispatch a `KeyboardEvent` for that key, verify callback was called once.
4. **preventDefault** -- dispatch a registered key event, check `event.defaultPrevented` is `true` on the returned event from `dispatchKey`.
5. **Unregistered key ignored** -- dispatch a key with no registration, verify no error and no callback fired.
6. **unregisterAll** -- register shortcuts, call `unregisterAll()`, verify `getRegistered()` returns empty array, dispatching the key no longer calls the callback, and `isEnabled()` is `true`.
7. **isEnabled = false disables dispatch** -- call `setEnabled(false)`, dispatch registered key, verify callback NOT called.
8. **isEnabled = true re-enables** -- disable then re-enable, verify dispatch works again.
9. **Re-registration replaces handler** -- register key 'a' with callback1, then register 'a' with callback2, dispatch 'a', verify only callback2 called (not callback1).
10. **Case normalization** -- register with uppercase key 'A', dispatch lowercase 'a', verify callback fires.
