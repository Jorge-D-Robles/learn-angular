# Plan: T-2026-430 — Create integration test for TerminalHackFormEvaluationService test case execution

## Task

Create integration tests for `TerminalHackFormEvaluationService` using **real level 1 data** from `TERMINAL_HACK_LEVELS[0]`. The existing unit spec (`terminal-hack-evaluation.service.spec.ts`) uses synthetic fixtures; this integration test catches **data-authoring bugs** — e.g., level data test cases that don't match the target spec, or element names that don't align with evaluation logic.

## Architecture

Pure test file — no production code changes, no TestBed.

### Why separate from the unit spec

The unit spec validates service logic with synthetic data. This integration spec validates that **real authored level data** works correctly through the service pipeline. It catches a class of bugs the unit spec cannot: mismatches between `targetFormSpec.elements[].name` and `testCases[].inputValues` keys, incorrect `expectedValid` values, tool/type combinations that don't evaluate as intended with real data.

### Level 1 data facts

From `TERMINAL_HACK_LEVELS[0]` (`th-basic-01`):
- `formType: 'template-driven'`
- One element: `id: 'name'`, `elementType: 'text'`, `name: 'crewName'`, no validations
- `availableElements: ['ngModel']`
- Test cases: tc1 (`crewName: 'Nova'`, expectedValid: true), tc2 (`crewName: ''`, expectedValid: true)

**Important field distinction:** The placed elements map is keyed by `FormElementSpec.id` (`'name'`). But `validateFormStructure` reports missing elements by `FormElementSpec.name` (`'crewName'`). Tests must use the correct field for each assertion.

### Data access

`TERMINAL_HACK_LEVELS[0]` is `LevelDefinition<TerminalHackLevelData>`. Level data is at `.data` — so `level1.data.targetFormSpec`, `level1.data.testCases`, etc.

### Helper: `makePlaced`

```typescript
function makePlaced(entries: [string, Partial<PlayerFormElement>][]): ReadonlyMap<string, PlayerFormElement> {
  return new Map(entries.map(([key, partial]) => [key, {
    elementId: partial.elementId ?? key,
    elementType: partial.elementType ?? 'text',
    toolType: partial.toolType ?? 'ngModel',
    validations: partial.validations ?? [],
  }]));
}
```

Note: `PlayerFormElement` uses `elementId` (not `id`). The map key and `elementId` should both match the `FormElementSpec.id`.

## Test Plan

### 1. "correct form passes all test cases"
- Place correct element: `makePlaced([['name', { elementType: 'text', toolType: 'ngModel' }]])`
- Call `service.runTestCases(placed, level1.data.testCases, level1.data.targetFormSpec)`
- Assert: `allPassed === true`, `passCount === 2`, `failCount === 0`, `passRate === 1.0`

### 2. "empty form fails all test cases"
- `placed = new Map()` (empty)
- Call `service.runTestCases(emptyPlaced, level1.data.testCases, level1.data.targetFormSpec)`
- Assert: `allPassed === false`, both test case results `passed === false`
- Assert: `testCaseResults[0].actualValid === false`, `testCaseResults[0].expectedValid === true`

### 3. "missing element fails structure validation"
- `placed = new Map()` (empty)
- Call `service.validateFormStructure(emptyPlaced, level1.data.targetFormSpec.elements)`
- Assert: `valid === false`
- Assert: `missingElements` contains `'crewName'` (the `FormElementSpec.name` field, NOT the `id` field `'name'`)
- Assert: `incorrectTypes` is empty

### 4. "extra validation causes test case failure"
- Place element with correct type/tool but add `required` validation: `makePlaced([['name', { validations: [{ type: 'required', errorMessage: 'Required' }] }]])`
- Call `service.runTestCases(placed, level1.data.testCases, level1.data.targetFormSpec)`
- tc1 (`crewName: 'Nova'`) passes — 'Nova' passes `required`
- tc2 (`crewName: ''`) fails — empty string fails `required`, so `actualValid: false` but `expectedValid: true`
- Assert: `passCount === 1`, `failCount === 1`, `passRate === 0.5`

### 5. "pass rate matches manual count"
- From test 4 result: `passRate === passCount / (passCount + failCount) === 1/2 === 0.5`
- Also verify: correct form → `passRate === 1.0`, empty form → `passRate === 0`
- Assert: `service.getTestPassRate()` matches the last `runTestCases` result's `passRate`

### 6. "evaluateForm returns per-element results with real data"
- Place correct element, call `service.evaluateForm(placed, level1.data.targetFormSpec)`
- Assert: 1 result, `correctType: true`, `correctTool: true`, `correctValidations: true`
