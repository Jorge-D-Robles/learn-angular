# Plan: T-2026-357 — Create LevelProgressSummaryComponent for minigame level completion counts

## Task

Navigation.md specifies the Minigame Level Select page shows "Best score / best time" per level and the Minigame Hub shows "Quick stats (levels completed, best scores)." Multiple pages need a compact display of level completion counts (e.g., "12/18 levels completed, 36 stars earned"). No reusable component exists for this.

Acceptance criteria:
- `LevelProgressSummaryComponent` at `src/app/shared/components/level-progress-summary/`
- Selector: `nx-level-progress-summary`
- Inputs: `completedLevels` (number), `totalLevels` (number), `totalStars` (number), `maxStars` (number)
- Displays: completion fraction (e.g., "12/18"), star total, progress bar
- Compact variant for card use, full variant for page headers

## Architecture

This is a pure presentational component with no service dependencies. It takes four numeric inputs and one variant selector, then renders a completion fraction, star count, and progress bar.

**Why a standalone component rather than a template snippet:** The MinigameHub page (`minigame-card` component) already shows `"12/18 levels"` inline, and the LevelSelectPage header will need similar display. A reusable component avoids duplication and ensures consistent formatting and accessibility across both contexts.

**Design decisions:**

1. **Reuse `ProgressBarComponent`** — The existing `nx-progress-bar` already handles percentage calculation, clamping, ARIA progressbar role, and variant styling. The summary component will compose it rather than reimplementing a progress bar.

2. **Two variants via `variant` input** — `'compact'` renders a single-line layout (fraction + stars inline, thin bar) for embedding in cards. `'full'` renders a stacked layout (larger text, thicker bar, explicit labels) for page headers.

3. **No lucide icons** — The star count uses the existing `LevelStarsComponent`? No: LevelStars renders 3 individual star icons for a single level's rating. Here we need a numeric star total (e.g., "36/54 stars"). We will render the star icon from lucide (`Star`) next to the numeric count for visual clarity.

4. **Edge cases** — `totalLevels === 0` renders "0/0" with 100% bar (same as ProgressBar behavior when max=0). Negative inputs are clamped to 0. `completedLevels > totalLevels` is clamped to `totalLevels`.

5. **Accessibility** — Host element gets `role="group"` with an `aria-label` summarizing the data (e.g., "Level progress: 12 of 18 levels completed, 36 of 54 stars earned"). The embedded progress bar provides its own ARIA progressbar semantics.

**Module boundaries:** This component lives in `src/app/shared/components/level-progress-summary/` alongside its spec and SCSS. It imports `ProgressBarComponent` and `LucideAngularModule` from existing shared code. It is exported via the components barrel.
