# Plan: T-2026-342 — Wire PhaseHeaderComponent and MissionCardComponent into CampaignPage

## Task
CampaignPage (T-2026-141) displays all 34 missions grouped by curriculum phase. PhaseHeaderComponent (T-2026-190) creates the phase grouping header with progress bar. MissionCardComponent (T-2026-216, completed) creates the mission entry card. This ticket wires PhaseHeaderComponent into CampaignPage's template, replacing the manual phase header markup with the reusable component. MissionCardComponent is already wired.

## Architecture
CampaignPage currently renders phase headers using raw HTML (`div.campaign__phase-header` with `h2`, `span`, `p`, and `nx-progress-bar`). PhaseHeaderComponent (`nx-phase-header`) already provides all of this functionality with collapsible accordion behavior, proper ARIA attributes, and content projection via `<ng-content>`.

The change is straightforward:
1. Add `PhaseHeaderComponent` to CampaignPage's `imports` array
2. Replace the `div.campaign__phase-header` block in the template with `<nx-phase-header>` and project the mission cards inside it
3. Remove `ProgressBarComponent` from imports since PhaseHeaderComponent replaces the per-phase progress bars. The overview section's progress bar still uses `ProgressBarComponent` directly.
4. Update tests to query for PhaseHeaderComponent elements instead of CSS classes

Note: MissionCardComponent is already imported and wired. No changes needed for it.
