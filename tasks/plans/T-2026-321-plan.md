## Architecture

This is a pure infrastructure/wiring ticket. No new logic or services are created.

**Pattern to follow:** Every other `src/app/core/*/index.ts` barrel uses `export * from './filename'` syntax, re-exporting all public symbols from each file in the subdirectory. The root `src/app/core/index.ts` then has one `export * from './subdirectory'` line per subdirectory.

**What to export from audio barrel:**
- `AudioService` (class, from `./audio.service`)
- `SoundEffect` (enum, from `./audio.service`)
- `SOUND_PATHS` (const, from `./audio.service`)

All three are already exported from `audio.service.ts` at the file level. The barrel just re-exports them.

**Existing import safety:** The only existing import of AudioService is in `audio.service.spec.ts`, which imports directly from `./audio.service`. This relative import will continue to work unchanged. No other file currently imports from audio. Downstream tickets (T-2026-322, T-2026-323, T-2026-325, T-2026-328) depend on T-2026-321 and will use the barrel.
