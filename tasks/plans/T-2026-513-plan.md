# Plan: T-2026-513 — Wire TerminalHackTestRunnerComponent into TerminalHackComponent UI

## Task

Wire the TerminalHackTestRunnerComponent child component into the parent TerminalHackComponent, replacing the inline bottom panel with the child component. Connect engine signals for test cases and results.

## Architecture

### Wiring Pattern

Follows T-2026-511 (CodePanel), T-2026-512 (LivePreview), T-2026-473 (FlowCommander) pattern.

### What Gets Replaced

The inline bottom panel (lines 46-68 of `terminal-hack.component.html`): Run Tests button, `@for` loop over test results, pass rate display. All replaced by `<app-terminal-hack-test-runner>`.

### Input Bindings

| Child Input | Bound To | Notes |
|---|---|---|
| `testCases` | `engine?.testCases() ?? []` | Direct template binding. `readonly FormTestCase[]` is assignable to `FormTestCase[]` in templates — no computed adapter needed. |
| `testResults` | `testRunResult()?.testCaseResults ?? null` | Must pass `null` (not `[]`) when no results exist. The child uses `!== null` to decide whether to show the pass-rate panel. The parent's existing `testResults` computed returns `[]` via `?? []`, which is WRONG for the child — it would show "0/0 tests passed" before any tests run. Bind directly in the template instead. |
| `isRunning` | `isTestsRunning()` | New local signal. See below. |

### `isRunning` Signal Strategy

The engine's `runTestCases()` is synchronous. Setting `isTestsRunning` to `true` then `false` in the same call frame means Angular change detection will never observe the `true` state — the animation won't fire.

**Solution:** Use `setTimeout(() => this.isTestsRunning.set(false), 0)` after the engine call. This lets Angular propagate the `true` state first (triggering the child's CSS animation), then clears it on the next microtask. The child's CSS `animation-fill-mode: forwards` ensures the animation completes visually even after the class is removed.

### Dead Code Removal

After wiring, these parent computed signals/methods become dead code (no template references):
- `canRunTests` — button disable logic moved to child
- `testResults` — replaced by direct template binding
- `allTestsPassed` — only used by removed inline display
- `passRatePercent` — only used by removed inline display

These should be **removed** from the component class.

**Keep:** `testRunResult` (used in the template binding `testRunResult()?.testCaseResults`), `onRunTests` (wired to output).
