# Plan: T-2026-024 -- Create StatePersistenceService for localStorage save/load

## Task

Service that persists game state to localStorage and restores it on app startup. All progression data must survive page refreshes.

Key capabilities: namespaced save/load/clear, graceful error handling for corrupted data, auto-save via `effect()` watching a signal, export/import for backup.

## Architecture

### Design Decisions

**Namespace prefix.** All localStorage keys are prefixed with `nexus-station:` to avoid collisions with other apps on the same origin. The prefix is defined as a private constant (`STORAGE_PREFIX`).

**Pure service, no coupling to GameStateService.** This service is a generic persistence layer. It knows nothing about game state shape -- it just serializes/deserializes JSON to/from namespaced localStorage keys. The integration with GameStateService is T-2026-025's responsibility. This keeps the service reusable for settings, streaks, leaderboards, play time, and daily challenges (all of which depend on this service per the backlog).

**`effect()` helper returns a cleanup function.** The `autoSave` method creates an Angular `effect()` that watches a provided signal and persists its value whenever it changes. It returns an `EffectRef` so the caller can manage the effect lifecycle (destroy when no longer needed). This uses Angular's `effect()` with `allowSignalWrites: false` (default) since it only reads signals, never writes them.

**Injector parameter for `autoSave`.** Since `effect()` must be called in an injection context (or be given an injector), the `autoSave` method accepts an optional `Injector` parameter. When called from a constructor or field initializer, no injector is needed. When called later (e.g., in `ngOnInit`), the caller passes their injector.

**Error handling strategy.** All `JSON.parse` failures are caught, log a `console.warn`, and return `null`. This means corrupted data is silently discarded rather than crashing the app. The warning is sufficient for debugging. `save()` also catches `JSON.stringify` errors (circular references) and `localStorage.setItem` errors (quota exceeded) with a `console.warn`.

**No versioning or migration in this ticket.** The ticket scope is basic persistence. Schema versioning and data migration are future concerns.

### Data Flow

```
Component/Service
    |
    v
StatePersistenceService.save(key, data)
    |
    v
JSON.stringify(data) -> localStorage.setItem("nexus-station:{key}", json)

StatePersistenceService.load<T>(key)
    |
    v
localStorage.getItem("nexus-station:{key}") -> JSON.parse(json) -> T | null
```

## Implementation Details

### `state-persistence.service.ts`

```ts
import { effect, Injectable, Injector, Signal, type EffectRef } from '@angular/core';

const STORAGE_PREFIX = 'nexus-station:';

@Injectable({ providedIn: 'root' })
export class StatePersistenceService {
  // --- Core operations ---

  save(key: string, data: unknown): boolean
  // Returns true on success, false on failure
  // Prefixes key with STORAGE_PREFIX
  // Catches JSON.stringify errors and quota exceeded errors

  load<T>(key: string): T | null
  // Returns parsed data or null if not found / corrupted
  // NOTE: null return is ambiguous — means either "key not found" OR "value was null"
  // Downstream callers should treat null as "no saved data" and use defaults
  // Catches JSON.parse errors, logs console.warn

  clear(key: string): void
  // Removes a single namespaced key

  clearAll(): void
  // Iterates localStorage keys, removes all starting with STORAGE_PREFIX

  // --- Auto-save ---

  autoSave<T>(key: string, source: Signal<T>, injector?: Injector): EffectRef
  // Creates an effect() that watches source and calls save(key, source()) on change
  // Returns EffectRef for lifecycle management
  // Accepts optional Injector for use outside injection context

  // --- Export / Import ---

  exportState(): string
  // Collects all nexus-station:* entries into a Record<string, unknown>
  // Keys in the output are the full prefixed keys (e.g., 'nexus-station:xp')
  // Values are JSON.parse'd from localStorage (so numbers stay numbers, etc.)
  // Returns JSON.stringify of that record

  importState(json: string): boolean
  // Parses json as Record<string, unknown>
  // Writes each entry to localStorage VERBATIM (keys already include prefix from export)
  // Does NOT apply _prefixedKey() -- avoids double-prefixing
  // Returns true on success, false on parse failure
}
```

### Key Implementation Notes

1. **`clearAll()` iteration**: Must iterate localStorage keys in reverse (or collect keys first) because removing keys during forward iteration skips entries. Collect matching keys into an array first, then remove each.

2. **`autoSave` uses `effect()`**: Angular's `effect()` runs once initially and then on signal changes. The first run will persist the initial value, which is the desired behavior (ensures localStorage is populated even before any mutations).

3. **`save()` returns boolean**: This allows callers (like the future auto-save integration in T-2026-025) to know if persistence succeeded, enabling retry or fallback logic.

4. **`importState()` does NOT call `clearAll()` first**: It merges/overwrites. The caller (settings reset UI) decides whether to clear first. This keeps the method composable. Each value is written via `localStorage.setItem(key, JSON.stringify(value))` to ensure proper serialization roundtrip with `exportState`.

5. **Private helper `_prefixedKey(key: string): string`**: Returns `${STORAGE_PREFIX}${key}` to DRY up the prefix logic.

## Test Plan

### `state-persistence.service.spec.ts`

Tests use a **Map-backed fake localStorage** injected via `Object.defineProperty(window, 'localStorage', { value: fakeStorage })`. This approach handles `length` (a property, not a method — cannot be `vi.spyOn`'d) and `key(i)` correctly. The fake implements: `getItem`, `setItem`, `removeItem`, `clear`, `key(i)`, and `length` getter.

```ts
function createFakeStorage(): Storage {
  const store = new Map<string, string>();
  return {
    getItem: (key: string) => store.get(key) ?? null,
    setItem: (key: string, value: string) => store.set(key, value),
    removeItem: (key: string) => store.delete(key),
    clear: () => store.clear(),
    key: (i: number) => [...store.keys()][i] ?? null,
    get length() { return store.size; },
  } as Storage;
}
```

For the `autoSave` test, use `TestBed.configureTestingModule({})` to get an injection context, `TestBed.inject(Injector)` for the injector parameter, and `TestBed.flushEffects()` after signal mutations to flush effect scheduling.

#### Test Cases

**1. save/load roundtrip**
- `save('profile', { name: 'Riker' })` then `load<{ name: string }>('profile')` returns `{ name: 'Riker' }`
- Verify `localStorage.setItem` was called with key `'nexus-station:profile'` and stringified JSON

**2. load returns null for missing key**
- `load('nonexistent')` returns `null`

**3. load returns null for corrupted JSON**
- Mock `localStorage.getItem` to return `'{invalid json'`
- `load('corrupted')` returns `null`
- Verify `console.warn` was called

**4. save handles stringify errors gracefully**
- Pass a value with circular reference (mock `JSON.stringify` to throw)
- `save('bad', circularObj)` returns `false`
- Verify `console.warn` was called

**5. save handles quota exceeded**
- Mock `localStorage.setItem` to throw `DOMException` (QuotaExceededError)
- `save('big', data)` returns `false`
- Verify `console.warn` was called

**6. clear removes a specific key**
- `save('a', 1)` then `clear('a')`
- Verify `localStorage.removeItem` was called with `'nexus-station:a'`

**7. clearAll removes only nexus-station keys**
- Set up localStorage mock with keys: `'nexus-station:a'`, `'nexus-station:b'`, `'other-app:c'`
- Call `clearAll()`
- Verify `removeItem` called for `'nexus-station:a'` and `'nexus-station:b'` but NOT `'other-app:c'`

**8. exportState returns JSON of all nexus-station entries**
- Set up localStorage mock with `'nexus-station:xp'` = `'100'` and `'nexus-station:name'` = `'"Picard"'`
- `exportState()` returns a JSON string that when parsed equals `{ 'nexus-station:xp': 100, 'nexus-station:name': 'Picard' }`

**9. importState restores entries**
- Call `importState('{"nexus-station:xp": 100, "nexus-station:name": "Picard"}')`
- Verify `localStorage.setItem` called for each entry with stringified values
- Returns `true`

**10. importState returns false on invalid JSON**
- `importState('not json')` returns `false`
- Verify `console.warn` was called

**11. autoSave creates an effect that persists on signal change**
- Use `TestBed.configureTestingModule({})` for injection context
- Get injector via `TestBed.inject(Injector)`
- Create a writable signal via `signal('initial')`
- Call `autoSave('test', sig, injector)` directly (the injector param handles the injection context — no need for `runInInjectionContext` wrapper)
- Call `TestBed.flushEffects()` — verify `save` was called with initial value
- Update the signal: `sig.set('updated')`
- Call `TestBed.flushEffects()` again — verify `save` was called with new value

**12. key namespacing**
- Verify all operations use the `nexus-station:` prefix (covered implicitly by the above tests, but one explicit test asserting the prefix format)

**13. load with valid but non-object JSON**
- `localStorage.getItem` returns `'"just a string"'`
- `load<string>('key')` returns `'just a string'` (primitives work too)

**14. save with null value**
- `save('key', null)` succeeds (saves `'null'`)
- `load('key')` returns `null` (which is indistinguishable from "not found" -- this is acceptable)
