# Plan: T-2026-124 — Wire top bar to display live rank and XP from services

## Task
The app shell top bar currently shows hardcoded "Cadet" rank and a static XP bar placeholder. Wire it to the real XpService to display live rank name and XP progress using the XpProgressBarComponent.

## Architecture

**Approach:** Inject `XpService` into the `App` component and replace the hardcoded rank text and static XP bar div with reactive signal bindings and the `<nx-xp-progress-bar>` component in compact variant.

**Key decisions:**

1. **Use `rankProgress()` signal directly** — XpService already exposes `rankProgress` (0–100 percentage). Pass it as `currentXp` with `nextRankXp=100` to XpProgressBarComponent. This avoids duplicating threshold math in the App component. The compact variant only shows the percentage bar (no raw XP text), so the semantic mismatch between "percentage passed as XP value" is invisible. At max rank, `rankProgress` returns 100, and the component computes `(100/100)*100 = 100%` — correct.

2. **Keep compact variant with percentage text** — The compact variant shows a small `NN%` label. This is useful context in the top bar. No CSS piercing needed (which would fail due to view encapsulation anyway).

3. **`currentRank` is a signal alias** — `readonly currentRank = this.xpService.currentRank;` is a `Signal<Rank>`. Call as `currentRank()` in the template.

4. **Settings button gets routerLink** — The ticket AC includes "Settings button navigates to `/settings` route via routerLink". Change the `<button>` to an `<a routerLink="/settings">` element. `RouterLink` is already imported in `app.ts`.

5. **`Rank` type is a string literal union** (`'Cadet' | 'Ensign' | ...`) — compatible with `input.required<string>()` on XpProgressBarComponent. No coercion needed.

## Detailed Changes

### `src/app/app.ts`

Add imports:
```typescript
import { Component, inject } from '@angular/core';
import { XpService } from './core/progression/xp.service';
import { XpProgressBarComponent } from './shared';
```

Add `XpProgressBarComponent` to `imports` array.

Add to component class:
```typescript
private readonly xpService = inject(XpService);

// Signal alias — call as currentRank() in template
readonly currentRank = this.xpService.currentRank;
readonly rankProgress = this.xpService.rankProgress;
```

### `src/app/app.html`

Replace line 5 (`<span class="top-bar__rank">Cadet</span>`) with:
```html
<span class="top-bar__rank">{{ currentRank() }}</span>
```

Replace lines 6-8 (the `.top-bar__xp` div with inner `.top-bar__xp-bar`) with:
```html
<nx-xp-progress-bar
  class="top-bar__xp"
  [currentXp]="rankProgress()"
  [nextRankXp]="100"
  [currentRank]="currentRank()" />
```

Note: `class="top-bar__xp"` on the host element applies the existing width constraint.

Replace the settings `<button>` (lines 11-13) with a link:
```html
<a class="top-bar__settings" routerLink="/settings" aria-label="Settings">
  <span class="top-bar__settings-icon" aria-hidden="true"></span>
</a>
```
