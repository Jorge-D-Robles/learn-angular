# Plan: T-2026-512 — Wire TerminalHackLivePreviewComponent into TerminalHackComponent UI

## Task

Wire the TerminalHackLivePreviewComponent child component into the TerminalHackComponent parent. Connect the engine's `placedElements()` signal to the live preview's `formElements` input, bind `targetSpec` from the engine, and forward the `elementClicked` output to `onSelectSlot`.

## Architecture

### Wiring Pattern

Follows the established P2/P4 wiring pattern (T-2026-511 CodePanel, T-2026-472 WireProtocol, T-2026-473 FlowCommander).

### Template Scoping Issue

The current template has:
```html
<div class="terminal-hack__content">
  @if (targetFormSpec(); as spec) {
    <app-terminal-hack-code-panel ... />
  }
  <div class="terminal-hack__sidebar">
    <!-- completion, hints -->
  </div>
</div>
```

The `spec` alias from `@if` is scoped to the `@if` block only — it is NOT available in the sibling sidebar div. The live preview's `targetSpec` input is `required`, so it needs a non-null value.

**Solution:** Wrap the live preview in its own `@if (targetFormSpec(); as previewSpec)` guard inside the sidebar. This is clean and independent — the sidebar still renders completion/hints even when `targetFormSpec()` is null, but the live preview only appears when there's a spec.

### Placement

The live preview goes at the **top of the sidebar**, before the completion indicator. This gives it visual prominence.

### Map-to-Array Conversion

The engine's `placedElements()` returns `ReadonlyMap<string, PlayerFormElement>`. The live preview expects `PlayerFormElement[]`. Add a new computed:
```typescript
readonly placedElementsArray = computed(() => Array.from(this.placedElements().values()));
```

### Output Forwarding

`(elementClicked)` → `onSelectSlot($event)`. This reactivates the dormant `onSelectSlot` method (removed from template in T-2026-511).

### No Template Removal

This ticket only **adds** the live preview. Nothing removed.
