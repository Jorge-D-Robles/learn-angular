## Architecture

This is a pure E2E test file -- no application code changes needed. The test follows the exact same pattern as `e2e/minigame-shell.spec.ts` (which already has a basic Terminal Hack render check) but goes deeper into verifying the Terminal Hack game's sub-components.

**Key design decisions:**

1. **Use `th-basic-01` as the test level** (not `/level/1`). The ticket description says `/level/1` but the actual routing uses level IDs like `th-basic-01`. The existing E2E test in `minigame-shell.spec.ts` already uses `th-basic-01`. This is the correct level ID for Terminal Hack's first level.

2. **Use component selectors and CSS class selectors, not test IDs.** The existing E2E tests use `page.locator('app-minigame-shell')` style selectors. We follow this convention. For HUD elements, we use the existing CSS classes (`.shell-hud__score`, `.shell-hud__hint`) and aria attributes.

3. **Verify presence, not behavior.** This is a smoke test. We confirm the game renders with all its panels visible, not that gameplay logic works. Gameplay is covered by unit/integration tests.

4. **The timer IS visible for `th-basic-01`.** The level data defines `timeLimit: 120`. The `TerminalHackEngine` overrides `config.timerDuration` to return `this._timeLimit()` (set from `data.timeLimit`). The shell's `showTimer` computed returns `timerDuration() > 0`, which is `true` for 120. So we CAN and SHOULD assert the timer element (`.shell-hud__timer`) is visible.

5. **Tutorial overlay may appear on first play.** The `MinigamePlayPage` checks `tutorialSeenKey` in persistence. In a fresh E2E context (no localStorage), the tutorial overlay will show if the game has tutorial steps. We need to handle dismissing the tutorial if it appears, or the game components may be behind the overlay. We should dismiss the tutorial first if it appears, then verify game elements.

- Create a helper function `dismissTutorialIfPresent(page)` that checks for the tutorial dialog and clicks the "Skip" button if visible. The tutorial is shown as a modal dialog, so it will block interaction with elements behind it until dismissed.

## Detailed Implementation

```typescript
// e2e/terminal-hack.spec.ts
import { test, expect, type Page } from '@playwright/test';

/**
 * Dismiss the minigame tutorial overlay if it appears.
 * Terminal Hack has 4 tutorial steps shown in a modal <dialog>.
 * Clicking "Skip" closes it immediately.
 */
async function dismissTutorialIfPresent(page: Page): Promise<void> {
  const skipBtn = page.locator('.tutorial__btn--skip');
  // Wait briefly for tutorial to appear (it fires in afterNextRender, so there's a short delay)
  try {
    await skipBtn.waitFor({ state: 'visible', timeout: 3000 });
    await skipBtn.click();
    await expect(skipBtn).not.toBeVisible();
  } catch {
    // Tutorial not present (e.g., already dismissed or no tutorial data) — continue
  }
}

test.describe('Terminal Hack', () => {
  test('should render game with target form preview and code editor', async ({ page }) => {
    await page.goto('/minigames/terminal-hack/level/th-basic-01');

    await expect(page.locator('app-minigame-shell')).toBeVisible();
    await expect(page.locator('app-terminal-hack')).toBeVisible();

    await dismissTutorialIfPresent(page);

    // Code panel with spec items and editor
    await expect(page.locator('app-terminal-hack-code-panel')).toBeVisible();
    await expect(page.locator('nx-code-editor')).toBeVisible();
    await expect(page.locator('.code-panel__spec-item').first()).toBeVisible();
  });

  test('should display MinigameShell HUD with score, timer, and hint button', async ({ page }) => {
    await page.goto('/minigames/terminal-hack/level/th-basic-01');

    await expect(page.locator('app-minigame-shell')).toBeVisible();
    await dismissTutorialIfPresent(page);

    // HUD elements
    await expect(page.locator('.shell-hud')).toBeVisible();
    await expect(page.locator('.shell-hud__score')).toBeVisible();
    await expect(page.locator('.shell-hud__timer')).toBeVisible();
    await expect(page.getByRole('button', { name: 'Pause' })).toBeVisible();

    // Terminal Hack's own hint button
    await expect(page.locator('.terminal-hack__hint-btn')).toBeVisible();
  });

  test('should render live preview panel with form slots', async ({ page }) => {
    await page.goto('/minigames/terminal-hack/level/th-basic-01');

    await expect(page.locator('app-terminal-hack')).toBeVisible();
    await dismissTutorialIfPresent(page);

    // Live preview panel
    await expect(page.locator('app-terminal-hack-live-preview')).toBeVisible();
    await expect(page.locator('.live-preview__header')).toBeVisible();
    await expect(page.locator('.live-preview__slot').first()).toBeVisible();
  });
});
```

**Tutorial dismiss confirmed:** The `MinigameTutorialOverlayComponent` at `src/app/shared/components/minigame-tutorial/minigame-tutorial.ts` renders a `<dialog>` with a "Skip" button (`.tutorial__btn--skip`). Clicking "Skip" calls `dismiss()` which closes the dialog and emits `dismissed`. The helper targets this exact button by its BEM class.
