# Plan: T-2026-288 — Add Page Visibility auto-pause to MinigameEngine

## Task

When a player switches browser tabs during timed gameplay, the engine timer continues counting down, which is unfair. The engine should auto-pause when the document becomes hidden and auto-resume when it becomes visible again. Manual pause (via pause menu) should NOT be auto-resumed -- only auto-pauses are auto-resumed.

## Architecture

### Key Design Decision: Document Access Without DI

MinigameEngine is a **plain abstract class**, not an `@Injectable`. It is instantiated directly via `new` (e.g., `new TestEngine(config)`) with no Angular injection context. The existing test suite uses direct instantiation without `TestBed`.

Using `inject(DOCUMENT)` would require either making the class injectable (too large a refactor for Size S) or requiring all consumers to wrap instantiation in an injection context (breaking change to tests and all subclasses).

**Chosen approach:** Accept an optional `document` reference in `MinigameEngineConfig`. When not provided, fall back to `typeof document !== 'undefined' ? document : undefined` for SSR safety. This keeps the class framework-agnostic and testable without DI, while allowing Angular components/services to pass their injected `DOCUMENT` token when constructing engines.

### Auto-Pause vs Manual Pause

The `_autoPaused` private boolean flag tracks whether the current pause was triggered by visibility change. The flag is:
- Set to `true` when `_onVisibilityChange` calls `pause()` (auto-pause)
- Set to `false` when `pause()` is called directly by game code (manual pause)
- Checked during `_onVisibilityChange` when document becomes visible: only resume if `_autoPaused === true`
- Reset to `false` on `resume()`, `start()`, `initialize()`, and `destroy()`

### Visibility Listener Lifecycle

- The `visibilitychange` listener is registered on `start()` (when the engine enters Playing state)
- The listener is removed on `destroy()`
- The listener is also removed on `initialize()` (since initialize resets all state and can be called from any state)
- The handler function is stored as a bound reference (`_boundVisibilityHandler`) for proper cleanup

### Data Flow

```
Tab hidden -> visibilitychange event -> _onVisibilityChange()
  -> if status === Playing: set _autoPaused = true, call pause()

Tab visible -> visibilitychange event -> _onVisibilityChange()
  -> if status === Paused && _autoPaused === true: set _autoPaused = false, call resume()
  -> if status === Paused && _autoPaused === false: no-op (manual pause)
```

## Detailed Changes

### `minigame-engine.ts`

1. **Add `document` to `MinigameEngineConfig`:**
   ```typescript
   export interface MinigameEngineConfig {
     readonly initialLives: number;
     readonly timerDuration: number | null;
     readonly document?: Document;
   }
   ```
   The `DEFAULT_ENGINE_CONFIG` does not include `document` (it remains optional/undefined).

2. **Add private fields** (alongside existing `_timerId`):
   ```typescript
   private _autoPaused = false;
   private readonly _boundVisibilityHandler: (() => void) | null;
   private readonly _doc: Document | undefined;
   ```
   **CRITICAL constructor ordering:** `_doc` and `_boundVisibilityHandler` must be resolved in the constructor body (NOT as field initializers), in this exact order:
   1. First: resolve `_doc` from merged config: `this._doc = mergedConfig.document ?? (typeof document !== 'undefined' ? document : undefined)`
   2. Then: set `_boundVisibilityHandler`: `this._boundVisibilityHandler = this._doc ? this._onVisibilityChange.bind(this) : null`

   **IMPORTANT:** `_doc` is stored as a separate private field, NOT as part of `_config`. The `_config` object exposed by the `config` getter must NOT include the `document` field — this avoids breaking the existing config getter test which asserts exact shape `{ initialLives: 3, timerDuration: null }`. Extract `document` from the merged config before storing `_config`:
   ```typescript
   const { document: configDoc, ...restConfig } = { ...DEFAULT_ENGINE_CONFIG, ...config };
   this._config = restConfig;
   this._doc = configDoc ?? (typeof document !== 'undefined' ? document : undefined);
   ```

3. **Add `_onVisibilityChange()` private method:**
   ```typescript
   private _onVisibilityChange(): void {
     if (!this._doc) return;
     if (this._doc.hidden && this._status() === MinigameStatus.Playing) {
       this._autoPaused = true;
       this.pause();
     } else if (!this._doc.hidden && this._status() === MinigameStatus.Paused && this._autoPaused) {
       this._autoPaused = false;
       this.resume();
     }
   }
   ```

4. **Add `_addVisibilityListener()` and `_removeVisibilityListener()` private methods:**
   ```typescript
   private _addVisibilityListener(): void {
     // Defensive remove-before-add to prevent double registration
     this._removeVisibilityListener();
     if (this._doc && this._boundVisibilityHandler) {
       this._doc.addEventListener('visibilitychange', this._boundVisibilityHandler);
     }
   }

   private _removeVisibilityListener(): void {
     if (this._doc && this._boundVisibilityHandler) {
       this._doc.removeEventListener('visibilitychange', this._boundVisibilityHandler);
     }
   }
   ```

5. **Modify `start()`:** Add `this._addVisibilityListener()` after `this._status.set(MinigameStatus.Playing)`.

6. **Modify `initialize()`:** Add `this._removeVisibilityListener()` and `this._autoPaused = false` at the top (before clearing timer), since initialize can be called from any state to reset.

7. **Modify `destroy()`:** Add `this._removeVisibilityListener()` and `this._autoPaused = false`.

8. **Modify `resume()`:** Add `this._autoPaused = false` (clearing the flag when resuming, whether auto or manual).

1. **"should auto-pause when document becomes hidden while Playing"**

   - Initialize, start, then trigger visibilitychange with `hidden = true`
   - Assert status is `Paused`

2. **"should auto-resume when document becomes visible after auto-pause"**

   - Assert status returns to `Playing`

3. **"should NOT auto-resume after manual pause"**

   - Assert status remains `Paused`

4. **"should not react to visibility changes when not Playing"**
   - Test that hidden event during Loading/Won/Lost/Paused(manual) states is a no-op

5. **"should clean up visibility listener on destroy"**
   - Verify `removeEventListener` was called with `'visibilitychange'` and the handler reference on destroy
   - After destroy, set hidden=true, dispatch — assert engine status remains unchanged (still the destroy state, no crash)

6. **"should clean up visibility listener on re-initialize"**
   - Verify `removeEventListener` was called when initialize is called mid-game
   - Then call `start()` again — verify listener is re-registered (addEventListener called again), and auto-pause works correctly

7. **"should pause timer during auto-pause"** (timer integration)
   - Use fake timers, start with timerDuration, auto-pause, advance time, assert timer did not decrement

### Mock Document Pattern for Tests

```typescript
function createMockDocument(): { hidden: boolean; addEventListener: Mock; removeEventListener: Mock; dispatchVisibilityChange: () => void } {
  let handler: (() => void) | null = null;
  const mockDoc = {
    hidden: false,
    addEventListener: vi.fn((event: string, fn: () => void) => {
      if (event === 'visibilitychange') handler = fn;
    }),
    removeEventListener: vi.fn((event: string, _fn: () => void) => {
      if (event === 'visibilitychange') handler = null;
    }),
    dispatchVisibilityChange() {
      handler?.();
    },
  };
  return mockDoc;
}
```

This mock captures the registered handler so tests can trigger it directly, and tracks add/remove calls for cleanup assertions.
