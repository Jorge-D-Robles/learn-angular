# Plan: T-2026-425 — Define cosmetic items content data for station skins, themes, and badges

## Task
Progression.md specifies "Cosmetic Unlocks (Future): Station module skins, UI themes, Achievement badges. Unlocked at rank milestones and mastery milestones." Define the content data catalog following the established `achievements.data.ts` pattern.

## Architecture
Pure data file with no Angular dependencies (no DI, no services). Follows the same pattern as `achievements.data.ts`:
- Type union for cosmetic categories
- `CosmeticDefinition` interface with all required fields
- `COSMETIC_DEFINITIONS` readonly array
- Convenience filtered views per type
- Barrel export from `src/app/data/index.ts`

Unlock conditions reference existing rank names from `src/app/core/state/rank.constants.ts` and achievement IDs from `achievements.data.ts` but do NOT import them — conditions are stored as human-readable strings (same pattern as `evaluationCriteria` in achievements).

## Interface Design

```typescript
export type CosmeticType = 'skin' | 'theme' | 'badge';

export interface CosmeticDefinition {
  readonly id: string;
  readonly name: string;
  readonly type: CosmeticType;
  readonly description: string;
  readonly unlockCondition: string;
  readonly previewImagePath?: string;
}
```
