# Plan: T-2026-052 — Create AnimationService for shared transition utilities

## Task

Visual style guide specifies animation principles: 150-250ms for UI transitions, 300-500ms for game feedback, ease-out for entrances, ease-in for exits, and respect for prefers-reduced-motion. This service provides reusable animation utilities and CSS animation classes that all components can use.

Acceptance criteria:
- `AnimationService` at `src/app/core/animation/animation.service.ts`
- `isReducedMotion` signal: reads from SettingsService and/or prefers-reduced-motion media query
- `ANIMATION_DURATIONS` constants: uiTransition (200ms), gameFeedback (400ms), overlay (300ms)
- Reusable animation classes at `src/app/core/animation/animations.ts` and `src/styles/_animations.scss`
- All animations use 0ms duration when reduced motion is active
- Unit tests for: reduced motion detection, animation constants

## Architecture

### Key Design Decision: CSS Animations Instead of @angular/animations

`@angular/animations` is **deprecated in Angular 21** and is **not installed** in this project. The codebase uses CSS animations and `@media (prefers-reduced-motion: reduce)` exclusively. The "animation triggers" will be implemented as **CSS `@keyframes` animations in a shared SCSS file** with corresponding `.nx-*` utility classes that components apply via standard Angular class bindings (`[class.nx-fade-in]="showItem"`).

### AnimationService Design

1. **`isReducedMotion` computed signal** — derives from `SettingsService.settings().reducedMotion`. The SettingsService already initializes `reducedMotion` from `window.matchMedia('(prefers-reduced-motion: reduce)')` at startup and allows the user to toggle it. No duplicated media query listener.

2. **`getDuration(key: AnimationDurationKey): number`** — returns 0 when reduced motion is active, otherwise returns the constant value. This is the programmatic hook for TypeScript code needing duration values (e.g., `setTimeout` delays). T-2026-211 will extend this to factor in `animationSpeed`.

### CSS Animation Approach

`_animations.scss` defines `@keyframes` rules and `.nx-*` utility classes. It uses the existing SCSS variables from `_tokens.scss` for durations and easings:
- `$nx-duration-fast: 150ms` → fadeIn/fadeOut
- `$nx-duration-normal: 250ms` → slideInRight/slideOutRight
- `$nx-duration-slow: 300ms` → scaleIn (overlay entry)
- `$nx-duration-game: 500ms` → pulse (closest to 400ms game feedback)

**Reduced motion in CSS:** The existing `_base.scss` already has a global `@media (prefers-reduced-motion: reduce)` block that sets `animation-duration: 0.01ms !important` for all elements. This global catch-all already handles CSS-level reduced motion. Therefore, `_animations.scss` does NOT add its own `@media (prefers-reduced-motion)` block — it relies on `_base.scss`. The `AnimationService.getDuration()` method handles the TypeScript-level reduced motion for programmatic delays.

### Constants File

`animations.ts` exports:
- `ANIMATION_DURATIONS` — semantic duration values for TypeScript callers (uiTransition, gameFeedback, overlay)
- `ANIMATION_CLASSES` — mapping of logical animation names to CSS class strings (fadeIn, fadeOut, slideInRight, slideOutRight, scaleIn, pulse)

## Detailed Implementation

### `animations.ts` — Constants

```typescript
export const ANIMATION_DURATIONS = {
  uiTransition: 200,
  gameFeedback: 400,
  overlay: 300,
} as const;

export type AnimationDurationKey = keyof typeof ANIMATION_DURATIONS;

export const ANIMATION_CLASSES = {
  fadeIn: 'nx-fade-in',
  fadeOut: 'nx-fade-out',
  slideInRight: 'nx-slide-in-right',
  slideOutRight: 'nx-slide-out-right',
  scaleIn: 'nx-scale-in',
  pulse: 'nx-pulse',
} as const;

export type AnimationClassName = keyof typeof ANIMATION_CLASSES;
```

`ANIMATION_DURATIONS` contains only the 3 semantic keys from the acceptance criteria. CSS-level durations are handled by `_tokens.scss` SCSS variables — no duplication.

### `animation.service.ts` — Service

```typescript
import { computed, inject, Injectable } from '@angular/core';
import { SettingsService } from '../settings/settings.service';
import { ANIMATION_DURATIONS, AnimationDurationKey } from './animations';

@Injectable({ providedIn: 'root' })
export class AnimationService {
  private readonly settings = inject(SettingsService);

  readonly isReducedMotion = computed(() => this.settings.settings().reducedMotion);

  getDuration(key: AnimationDurationKey): number {
    return this.isReducedMotion() ? 0 : ANIMATION_DURATIONS[key];
  }
}
```

### `_animations.scss` — CSS Animations

Must `@use 'tokens' as t;` and reference existing SCSS variables:

```scss
@use 'tokens' as t;

// --- Keyframes ---

@keyframes nx-fade-in-kf {
  from { opacity: 0; }
  to { opacity: 1; }
}

@keyframes nx-fade-out-kf {
  from { opacity: 1; }
  to { opacity: 0; }
}

@keyframes nx-slide-in-right-kf {
  from { transform: translateX(100%); opacity: 0; }
  to { transform: translateX(0); opacity: 1; }
}

@keyframes nx-slide-out-right-kf {
  from { transform: translateX(0); opacity: 1; }
  to { transform: translateX(100%); opacity: 0; }
}

@keyframes nx-scale-in-kf {
  from { transform: scale(0.8); opacity: 0; }
  to { transform: scale(1); opacity: 1; }
}

@keyframes nx-pulse-kf {
  0%, 100% { transform: scale(1); }
  50% { transform: scale(1.05); }
}

// --- Utility classes ---

.nx-fade-in {
  animation: nx-fade-in-kf t.$nx-duration-fast t.$nx-ease-out both;
}

.nx-fade-out {
  animation: nx-fade-out-kf t.$nx-duration-fast t.$nx-ease-in both;
}

.nx-slide-in-right {
  animation: nx-slide-in-right-kf t.$nx-duration-normal t.$nx-ease-out both;
}

.nx-slide-out-right {
  animation: nx-slide-out-right-kf t.$nx-duration-normal t.$nx-ease-in both;
}

.nx-scale-in {
  // Ticket AC says 200ms; closest token is $nx-duration-normal (250ms).
  // Using 200ms literal since no exact SCSS variable exists. If a $nx-duration-200 token is added later, switch to it.
  animation: nx-scale-in-kf 200ms t.$nx-ease-out both;
}

.nx-pulse {
  animation: nx-pulse-kf t.$nx-duration-game ease-in-out;
}
```

**Note:** No `@media (prefers-reduced-motion)` block — `_base.scss` already handles this globally.
