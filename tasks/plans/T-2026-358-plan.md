# Plan: T-2026-358 — Create integration test for SettingsService reset triggering StatePersistenceService clearAll

## Task

Architecture.md documents that "SettingsService.resetProgress() calls StatePersistenceService.clearAll() followed by a full page reload." No integration test verifies this critical destructive flow works correctly end-to-end.

Acceptance criteria:
- Integration test at `src/app/core/integration/settings-reset.integration.spec.ts`
- Test: add game state (XP, mastery, streaks), call resetProgress(), verify all localStorage cleared
- Test: verify resetProgress() triggers page reload (document.location.reload)
- Test: verify all services would initialize with defaults after clear (fresh state)
- Uses real StatePersistenceService with fake localStorage

## Architecture

This is a focused integration test that verifies the contract between `SettingsService` and `StatePersistenceService` in the context of the "reset progress" destructive action. The test uses **real service instances** (no mocks on the persistence layer) backed by a **fake localStorage** (the Map-backed `createFakeStorage()` pattern established throughout the codebase). The only mock is `DOCUMENT`, which captures the `location.reload()` call without actually reloading.

**Why this approach:**
- Using real `StatePersistenceService` with fake localStorage is the established pattern in every integration test in `src/app/core/integration/`. This gives us genuine end-to-end verification that `clearAll()` actually removes prefixed keys.
- Mocking `DOCUMENT` is necessary because `location.reload()` would crash the test runner. The existing `settings.service.spec.ts` already uses this exact pattern (lines 269-294).
- We seed multiple `nexus-station:*` keys to simulate real game state (XP, mastery, streaks, level progression, settings) and also seed a non-prefixed key to verify `clearAll()` only removes namespaced keys.
- A "fresh state after clear" test re-creates the TestBed after clearing storage, injects `StatePersistenceService`, and verifies `load()` returns null for all game keys -- proving that any service re-initialized after the reload would get defaults.

**Data flow under test:**
1. Seed fake localStorage with multiple `nexus-station:*` keys (simulating accumulated game state)
2. Call `SettingsService.resetProgress()`
3. This calls `StatePersistenceService.clearAll()` (iterates localStorage, removes all `nexus-station:*` keys)
4. Then calls `document.location.reload()`
5. Verify all `nexus-station:*` keys are gone, non-prefixed keys survive, and reload was called

## Test Plan

The spec file contains one `describe` block with four tests:

### Test 1: "should clear all nexus-station localStorage keys when resetProgress is called"
- **Setup:** Seed fake localStorage with 5 `nexus-station:*` keys (`game-state`, `mastery`, `streak`, `level-progression`, `settings`) containing realistic JSON data. Also seed one non-prefixed key (`other-app:prefs`).
- **Action:** Call `settingsService.resetProgress()`.
- **Assert:** All 5 `nexus-station:*` keys return `null` from `fakeStorage.getItem()`. The `other-app:prefs` key is still present. `fakeStorage.length` equals 1 (only the non-prefixed key remains).

### Test 2: "should trigger page reload via document.location.reload"
- **Setup:** Standard setup with fake DOCUMENT.
- **Action:** Call `settingsService.resetProgress()`.
- **Assert:** `fakeDoc.location.reload` has been called exactly once.

### Test 3: "should call clearAll before reload"
- **Setup:** Spy on `StatePersistenceService.clearAll` and capture call order relative to `fakeDoc.location.reload`.
- **Action:** Call `settingsService.resetProgress()`.
- **Assert:** `clearAll` was called before `reload` (verify via call order tracking with a shared array that both spies push to).

### Test 4: "should result in null loads for all game keys after clear (fresh state)"
- **Setup:** Seed fake localStorage with game state keys. Call `settingsService.resetProgress()`.
- **Action:** Use `StatePersistenceService.load()` to attempt loading each key.
- **Assert:** All calls return `null`, proving that any service re-initialized after the reload would fall back to defaults.
