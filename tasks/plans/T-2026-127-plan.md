# Plan: T-2026-127 -- Wire LevelCompletionService to XpNotificationService

## Task

LevelCompletionService orchestrates the score->progress->XP->mastery flow, and XpNotificationService shows XP toast notifications. These two services are not connected -- completing a level awards XP but does not trigger a visible notification. This ticket wires them together so players see XP gain feedback.

## Architecture

**Approach: Direct injection.** LevelCompletionService already injects three services (LevelProgressionService, XpService, MasteryService). Adding XpNotificationService as a fourth injected dependency is the simplest, most direct approach. It keeps the notification call co-located with the rank-up detection logic, which is needed to include rank-up info in the notification.

**Why not an event/Observable approach?** An event bus or Observable emission would decouple the services but adds indirection for no benefit -- there is exactly one consumer (XpNotificationService) and the notification must fire synchronously after XP is awarded and rank-up is detected. Direct injection avoids subscription management and is consistent with how the service already coordinates its three dependencies.

**Notification content:** The `XpNotificationService.show()` API accepts `amount: number` and `bonuses: readonly string[]`. The bonuses array will carry contextual labels:
- Always: `"Level Complete"` (the source label)
- If `result.perfect === true`: `"Perfect!"`
- If `streakMultiplier > 1.0`: `"Streak Bonus"`
- If `rankUpOccurred`: `"Rank Up: <NewRankName>"` (e.g., `"Rank Up: Ensign"`)

This uses the existing API without modification. The XP amount is already computed before the notification call.

## Detailed Changes

### `level-completion.service.ts`

1. Add import for `XpNotificationService` from `../notifications` (barrel export, consistent with project convention).
2. Add `private readonly xpNotification = inject(XpNotificationService);` to the class body.
3. After the return object is constructed (after step 8, before `return`), build the bonuses array and call `this.xpNotification.show()`:

```typescript
// 9. Trigger XP notification
const bonuses: string[] = ['Level Complete'];
if (result.perfect === true) {
  bonuses.push('Perfect!');
}
if (streakMultiplier > 1.0) {
  bonuses.push('Streak Bonus');
}
if (rankUpOccurred) {
  bonuses.push(`Rank Up: ${rankAfter}`);
}
this.xpNotification.show(xpEarned, bonuses);
```

This is placed before the `return` statement so all data (`xpEarned`, `rankUpOccurred`, `rankAfter`, `streakMultiplier`, `result.perfect`) is already computed.

**Note on `rankAfter`:** It is typed as `Rank` (a `string` union: `'Cadet' | 'Ensign' | ...`), so the template literal `` `Rank Up: ${rankAfter}` `` requires no additional coercion.

### `level-completion.service.spec.ts`

Add a new `describe('XP notifications')` block with these tests:

1. **"should trigger XP notification on level completion"** -- Spy on `XpNotificationService.show`, call `completeLevel()`, assert `show` was called once.
2. **"should pass correct XP amount to notification"** -- Complete a Basic non-perfect level (15 XP), assert `show` was called with `amount = 15`.
3. **"should include 'Level Complete' in bonuses"** -- Complete any level, assert bonuses array contains `'Level Complete'`.
4. **"should include 'Perfect!' bonus when perfect"** -- Complete with `perfect: true`, assert bonuses includes `'Perfect!'`.
5. **"should not include 'Perfect!' bonus when not perfect"** -- Complete with `perfect: false`, assert bonuses does not include `'Perfect!'`.
6. **"should include 'Streak Bonus' when streak multiplier > 1"** -- Complete with `{ streakMultiplier: 1.3 }`, assert bonuses includes `'Streak Bonus'`.
7. **"should include rank-up info in notification when rank changes"** -- Set XP to 490 (Cadet), complete with `perfect: true` (+30 XP -> 520 -> Ensign), assert bonuses includes `'Rank Up: Ensign'`.
8. **"should not include rank-up info when rank stays same"** -- Complete at 0 XP, assert no bonus string starts with `'Rank Up'`.

**Spy setup:** Use inline spy per test (matching existing codebase style — see mastery spy at line 256 of the spec). Create the spy at the start of each test: `const spy = vi.spyOn(TestBed.inject(XpNotificationService), 'show');`

**Timer safety:** The `show()` method creates a `setTimeout` internally. Add `vi.useFakeTimers()` in the `describe('XP notifications')` `beforeEach` and `vi.useRealTimers()` in its `afterEach` to prevent dangling timers between tests.
