## Task
XpNotificationComponent and XpNotificationService were built (T-2026-032) but the notification component is not rendered anywhere in the app shell. It needs to be added at the root level so XP toast notifications appear globally when XP is awarded from any page.

## Architecture
This is a pure integration task -- no new components or services. The XpNotificationComponent already exists at `src/app/shared/components/xp-notification/xp-notification.ts` with selector `nx-xp-notification`. The XpNotificationService is `providedIn: 'root'` so no provider registration is needed.

The component must be placed **after** the `.app-body` div in `app.html` so that toast notifications render in a layer above all routed content. This placement (outside the main layout flow) is correct for overlay/toast UI -- it avoids being clipped by any overflow rules on `.app-body` or `.content`.

The import path will use the barrel export from `src/app/shared` which already re-exports `XpNotificationComponent`.
