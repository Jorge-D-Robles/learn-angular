# Plan: T-2026-193 --- Create Terminal Hack minigame tutorial data

## Task

Create tutorial step data for the Terminal Hack minigame so `MinigameTutorialOverlayComponent` can display first-time play instructions. Add the data to `src/app/data/tutorials/minigame-tutorials.data.ts` following the existing P2 pattern. 3-4 tutorial steps covering: target form reading, code editor usage, live preview, test runner, hint system.

## Architecture

This is a content-only addition to an existing data file. No new modules, services, or components are needed.

**Pattern:** The existing file `src/app/data/tutorials/minigame-tutorials.data.ts` defines a `const` array of `TutorialStep[]` per minigame, then adds it to the exported `MINIGAME_TUTORIALS` array. Each step has `title` (short heading) and `description` (1-2 sentence explanation). The `getMinigameTutorial()` lookup function already handles any `MinigameId`, so no changes are needed there.

**Decision -- 4 steps:** The ticket says 3-4 steps. Terminal Hack has 5 distinct mechanics listed in the acceptance criteria (target form reading, code editor usage, live preview, test runner, hint system). Four steps is the right count: combine "target form reading" and "live preview" into one step (they are complementary -- the player reads the target on the left and sees their progress on the right). This gives us:

1. **Read the Target Form** -- Explains the left panel (target preview) and right panel (live preview) split, and the goal of matching the target.
2. **Write Form Code** -- Explains the code editor where the player writes template + component class code.
3. **Run Tests** -- Explains the test button that runs predefined input/output cases against the form. Also mentions the timer (power gauge) that adds urgency and the lose condition (failing 3+ tests or running out of time).
4. **Use Hints** -- Explains the hint button that reveals one form element at a cost to score.

This mirrors the design doc's "Controls" section (code editor, test button, submit button, hint button) and covers all five acceptance criteria topics within the 4-step budget.

**Why not 3 steps:** Collapsing further would lose clarity. The hint system is a distinct strategic mechanic (costs points) that deserves its own step.

**Why not 5 steps:** The ticket caps at 4. Target form reading and live preview are two sides of the same panel layout and belong together.
