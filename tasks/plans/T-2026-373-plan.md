# Plan: T-2026-373 -- Integration test for DailyChallengeService topic rotation prioritizing degrading topics

## Task
Create an integration test verifying that DailyChallengeService prioritizes degrading topics from SpacedRepetitionService in the daily challenge rotation. Uses real services (not mocks) to verify the end-to-end degradation priority algorithm.

## Architecture
The test exercises the real interaction chain:
- MasteryService holds raw mastery stars for topics
- SpacedRepetitionService tracks lastPracticed timestamps and computes degradation
- GameProgressionService tracks which minigames are unlocked (via completed missions)
- DailyChallengeService calls `spacedRepetition.getDegradingTopics()` and `gameProgression.getUnlockedMinigames()` to select the daily game

The integration test will set up real services with controlled state (via fake localStorage) to verify:
1. With no degraded topics, challenge selects from the general unlocked pool
2. With degraded topics, challenge preferentially selects the degraded topic's minigame
3. Rotation does not repeat the same game on consecutive days (when alternatives exist)

Key insight: DailyChallengeService freezes its challenge at construction time (constructor calls `_generateChallenge`). To test different days, we must reset TestBed and re-inject for each simulated day.

SpacedRepetitionService.getDegradingTopics() requires:
- A topic in `_lastPracticed` map (recorded via `recordPractice()`)
- The topic's raw mastery > 0 (from MasteryService)
- daysSinceLastPractice >= 7 (grace period)

GameProgressionService.getUnlockedMinigames() requires:
- Completed missions that have `unlocksMinigame` set

So to get a "real" degrading topic without mocking, we must:
1. Seed mastery data into localStorage (so MasteryService loads non-zero mastery)
2. Seed spaced-repetition data with a lastPracticed timestamp > 7 days ago
3. Seed game-progression data with completed mission chapter IDs that unlock the relevant minigames

## Test Plan

- Seed localStorage with: mastery data (stars for some topics), spaced-repetition data with recent practice (within 7 days), and completed missions unlocking multiple minigames
- Construct DailyChallengeService
- Assert: todaysChallenge().gameId is one of the unlocked minigames

### Test 2: With degraded topics, daily challenge preferentially selects degraded topic's minigame
- Seed localStorage with: mastery data (stars for some topics), spaced-repetition data with old practice (>7 days ago) for one specific topic, and completed missions unlocking multiple minigames
- Construct DailyChallengeService
- Assert: todaysChallenge().gameId equals the degrading topic's minigame ID

### Test 3: Rotation does not repeat the same game on consecutive days (when alternatives exist)
- Seed state so that multiple degrading topics exist (at least 2)
- Simulate multiple consecutive days by advancing fake timers and re-creating TestBed
- Collect the gameId for each day
- Assert: not all gameIds are the same (at least one differs from the others across enough days)
