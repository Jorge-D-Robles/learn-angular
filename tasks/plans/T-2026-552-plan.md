# Plan: T-2026-552 — Integrate MinigameEngine rendering into EndlessModePage for actual gameplay

## Task

EndlessModePage has UI shells for pre-game/in-game/post-game states and tracks high scores, but the `in-game` state does not actually render a MinigameEngine or its UI component. Players cannot play actual minigame rounds in endless mode. The page needs to instantiate the correct engine via MinigameRegistryService, render the game-specific UI component via NgComponentOutlet, and feed procedurally generated levels from EndlessModeService.

## Architecture

### Core Problem

The current EndlessModePage `in-game` state renders static text (round number, score, difficulty label, "End Session" button) but never instantiates a MinigameEngine, never creates an `Injector` with the `MINIGAME_ENGINE` token, and never renders the game-specific component via `NgComponentOutlet`. The MinigamePlayPage already solves this problem for story mode; we adapt that pattern for endless mode.

### Key Design Decisions

1. **Procedural level generation via `EndlessModeService.generateLevel()`**: The service already has `getDifficultyParams(round)` returning speed/complexity/count scaling. We add a `generateLevel(gameId, round)` method that produces a `MinigameLevel<unknown>` with synthetic data shaped by those difficulty params. The level data will be a generic `EndlessLevelData` object containing the difficulty params -- individual engines can interpret these params via their `onLevelLoad()` implementations. This is the simplest approach that works across all 12 minigame types without requiring per-game procedural generators.

2. **Engine lifecycle mirrors MinigamePlayPage pattern**: Use `NgComponentOutlet` + custom `Injector` providing `MINIGAME_ENGINE` token, exactly as MinigamePlayPage does. The MinigameShellComponent wraps the game component and provides the HUD (score, lives, timer).

3. **Round progression via engine status effects**: An Angular `effect()` watches `engine.status()`. When the engine transitions to `Won`, the page calls `endlessModeService.nextRound(score)`, generates the next level, re-initializes and restarts the engine. When the engine transitions to `Lost`, the session ends and transitions to post-game. **Note**: The effect will re-fire on intermediate states (`Loading`, `Playing`) during round transitions, but these are no-ops since only `Won` and `Lost` trigger action.

4. **PlayMode.Endless**: The engine is initialized with `PlayMode.Endless` so individual engines can adapt behavior (e.g., no level-completion XP recording).

5. **No LevelLoaderService dependency**: Unlike MinigamePlayPage which loads levels from JSON files, EndlessModePage generates levels procedurally -- no async loading needed.

6. **Partial score accounting**: When the player loses a round, the score earned in that partial round IS included in the final session total. `nextRound(partialScore)` is called before `endSession()` so the session accumulates the partial round's score.

### Why This Approach Over Alternatives

- **Alternative: Reuse MinigamePlayPage with a mode flag** -- Rejected because the lifecycle is fundamentally different (no level loader, no level progression service, continuous round cycling, different completion semantics). Sharing would create a god component.
- **Alternative: Per-game procedural generators** -- Over-engineered for now. The generic `EndlessLevelData` with difficulty params lets engines interpret difficulty however they want. Per-game generators can be added later as a separate ticket.

## Detailed Changes

### 1. EndlessModeService: Add `generateLevel()`

Add an exported interface and a new method:

```typescript
/** Level data shape for procedurally generated endless mode rounds. */
export interface EndlessLevelData {
  readonly round: number;
  readonly difficulty: DifficultyParams;
}
```

New method on `EndlessModeService`:

```typescript
generateLevel(gameId: MinigameId, round: number): MinigameLevel<EndlessLevelData> {
  const difficulty = this.getDifficultyParams(round);
  return {
    id: `endless-${gameId}-r${round}`,
    gameId,
    tier: this._tierForRound(round),
    conceptIntroduced: 'Endless Mode',
    description: `Endless round ${round}`,
    data: { round, difficulty },
  };
}
```

Private helper `_tierForRound(round)`:

```typescript
private _tierForRound(round: number): DifficultyTier {
  if (round <= 3) return DifficultyTier.Basic;
  if (round <= 6) return DifficultyTier.Intermediate;
  if (round <= 9) return DifficultyTier.Advanced;
  return DifficultyTier.Boss;
}
```

**New signals/computed:**

```typescript
private readonly parentInjector = inject(Injector);

engine = signal<MinigameEngine<unknown> | null>(null);

resolvedComponent = computed(() => {
  const id = this.gameId();
  if (!id) return null;
  return this.registry.getComponent(id as MinigameId) ?? null;
});

engineInjector = computed(() => {
  const eng = this.engine();
  if (!eng) return null;
  return Injector.create({
    providers: [{ provide: MINIGAME_ENGINE, useValue: eng }],
    parent: this.parentInjector,
  });
});
```

**Note on `engineInjector` stability**: The engine signal reference stays the same between rounds (we reuse the engine instance via `eng.initialize()` + `eng.start()`). The injector computed only re-creates when `engine()` identity changes (initial creation and cleanup), NOT on every round transition.

Replace the current static text with:

**Modified `onStart()` method:**

```typescript
onStart(): void {
  const id = this.gameId();
  if (!id) return;

  const factory = this.registry.getEngineFactory(id as MinigameId);
  if (!factory) return;

  // Guard: check factory exists BEFORE starting session to avoid orphaned sessions
  this.endlessModeService.startSession(id as MinigameId);
  const eng = factory();
  const level = this.endlessModeService.generateLevel(id as MinigameId, 1);
  eng.initialize(level);
  eng.setPlayMode(PlayMode.Endless);
  eng.start();
  this.engine.set(eng);
  this.viewState.set('in-game');
}
```

**New effect for round cycling / failure detection:**

```typescript
// In constructor:
effect(() => {
  const eng = this.engine();
  if (!eng) return;
  const status = eng.status();

  // Only act on terminal states. Intermediate states (Loading, Playing) are no-ops.
  untracked(() => {
    if (status === MinigameStatus.Won) {
      this.onRoundWon();
    } else if (status === MinigameStatus.Lost) {
      this.onRoundLost();
    }
  });
});
```

**New methods:**

```typescript
private onRoundWon(): void {
  const eng = this.engine();
  const session = this.session();
  if (!eng || !session?.isActive) return;

  // Capture score BEFORE re-initializing (initialize() resets _score to 0)
  const roundScore = eng.score();
  this.endlessModeService.nextRound(roundScore);

  const updatedSession = this.endlessModeService.session();
  if (!updatedSession) return;

  const nextLevel = this.endlessModeService.generateLevel(
    updatedSession.gameId,
    updatedSession.currentRound,
  );
  eng.initialize(nextLevel);
  eng.setPlayMode(PlayMode.Endless);
  eng.start();
}

private onRoundLost(): void {
  const eng = this.engine();
  const session = this.session();
  if (!session?.isActive) return;

  // Include partial score from the failed round in the session total
  const partialScore = eng?.score() ?? 0;
  if (partialScore > 0) {
    this.endlessModeService.nextRound(partialScore);
  }

  const { finalScore, isNewHighScore } = this.endlessModeService.endSession();
  const roundsSurvived = session.currentRound;

  // Set engine to null BEFORE setting postGameData to avoid effect re-triggering
  this.engine.set(null);

  this.postGameData.set({ finalScore, roundsSurvived, isNewHighScore });
  this.viewState.set('post-game');
}

onPause(): void {
  this.engine()?.pause();
}

onResume(): void {
  this.engine()?.resume();
}

onRestartRound(): void {
  const eng = this.engine();
  const session = this.session();
  if (!eng || !session) return;

  // Do NOT use engine.reset() — it calls initialize() + start() without setPlayMode(),
  // which would revert to PlayMode.Story. Instead, explicitly re-initialize.
  const level = this.endlessModeService.generateLevel(
    session.gameId,
    session.currentRound,
  );
  eng.initialize(level);
  eng.setPlayMode(PlayMode.Endless);
  eng.start();
}
```

**Modified `onEndSession()`:**

```typescript
onEndSession(): void {
  // Set engine to null FIRST to prevent the status-watching effect from re-triggering
  const eng = this.engine();
  this.engine.set(null);
  eng?.destroy();

  const session = this.session();
  if (session?.isActive) {
    const { finalScore, isNewHighScore } = this.endlessModeService.endSession();
    this.postGameData.set({ finalScore, roundsSurvived: session.currentRound, isNewHighScore });
  }
  this.viewState.set('post-game');
}
```

**Modified `onPlayAgain()`:**

```typescript
onPlayAgain(): void {
  // Engine should already be null from onEndSession()/onRoundLost(), but guard
  if (this.engine()) {
    const eng = this.engine();
    this.engine.set(null);
    eng?.destroy();
  }
  this.postGameData.set(null);
  this.viewState.set('pre-game');
}
```

**Cleanup on destroy via DestroyRef:**

```typescript
// In constructor:
inject(DestroyRef).onDestroy(() => {
  const eng = this.engine();
  if (eng) {
    this.engine.set(null);
    eng.destroy();
  }
});
```
