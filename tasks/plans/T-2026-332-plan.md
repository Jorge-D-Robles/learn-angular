# Plan: T-2026-332 — Create P2 cross-cutting integration test for story mission to minigame unlock flow

## Task

Create an integration test that verifies the core game loop defined in `docs/overview.md`: "Complete Story Mission -> Unlock Minigame -> Master Minigame -> Earn XP -> Level Up." The test file lives at `src/app/features/integration/story-to-minigame.integration.spec.ts` and exercises real `CurriculumService`, `GameProgressionService`, `StoryMissionCompletionService`, `MasteryService`, `XpService`, and `LevelCompletionService` instances wired together in TestBed.

Acceptance criteria:
- Integration test at `src/app/features/integration/story-to-minigame.integration.spec.ts`
- Test: completing story mission 1 unlocks Module Assembly minigame
- Test: completing story mission unlocks the correct minigame per curriculum mapping
- Test: locked minigame cannot be played before its prerequisite story mission
- Test: XP is awarded for both story mission completion and minigame completion
- Uses real CurriculumService, ProgressionService, and level data

## Architecture

### Why this test exists

T-2026-332 and T-2026-403 are related but distinct:
- **T-2026-332 (this ticket)** focuses on the **story mission -> minigame unlock** cross-cutting boundary. It verifies the curriculum mapping, the lock/unlock gate, and that XP flows from both story and minigame completions. It does NOT instantiate actual minigame engines.
- **T-2026-403** focuses on the **full game loop pipeline** end-to-end, including engine initialization, level data loading, and diminishing returns. It is a deeper vertical slice.

This test sits in `src/app/features/integration/` (alongside `spaced-repetition-alert.integration.spec.ts`) because it crosses feature boundaries (curriculum, progression, minigame unlock).

### Key design decisions

1. **Use real `providedIn: 'root'` services via empty `TestBed.configureTestingModule({})`.** All services under test (`GameProgressionService`, `StoryMissionCompletionService`, `CurriculumService`, `XpService`, `MasteryService`, `LevelCompletionService`, `LevelProgressionService`, `XpDiminishingReturnsService`) are `providedIn: 'root'`, so TestBed provides them automatically. This matches the pattern in `mission-completion.integration.spec.ts`.

2. **Mock `Audio` globally.** `GameProgressionService` -> `MissionUnlockNotificationService` -> `AudioService` requires `Audio`. Use the `MockAudio` pattern established in `mission-completion.integration.spec.ts` and `level-completion-pipeline.integration.spec.ts`.

3. **Fake `localStorage`.** All persistence services use `localStorage` via `StatePersistenceService`. Use the `createFakeStorage()` pattern from existing tests.

4. **Mock `matchMedia`.** `SettingsService` (injected transitively by `AudioService`) calls `window.matchMedia` for theme detection. Mock it as in existing tests.

5. **Use `vi.useFakeTimers()`.** Services use debounced saves and streak calculations that depend on `Date.now()`. Fake timers give deterministic behavior.

6. **Register test level data manually.** For the "XP from minigame level" test, register a small `LevelDefinition[]` array via `LevelProgressionService.registerLevels()` and call `LevelCompletionService.completeLevel()` with a `MinigameResult`. This avoids importing the full level data packs and keeps the test focused.

7. **Curriculum mapping verification uses `CurriculumService.getChapterForMinigame()` and `ALL_STORY_MISSIONS` data.** The test verifies specific chapter-to-minigame relationships from the curriculum data (Ch 1 -> module-assembly, Ch 4 -> flow-commander, Ch 5 -> wire-protocol, Ch 7 -> signal-corps).

### Data flow under test

```
StoryMissionCompletionService.completeMission(chapterId)
  -> GameProgressionService.completeMission(chapterId)
       -> _completedMissions signal updated
       -> XpService.calculateStoryXp() -> 50 XP
       -> XpService.applyStreakBonus() -> XpWithStreakBonus
       -> XpService.addXp() -> GameStateService.addXp()
       -> MissionUnlockNotificationService.showUnlock() [if mission.unlocksMinigame]
  -> CurriculumService.getMinigameForChapter(chapterId) -> MinigameId | null
  -> MasteryService.ensureMinimumMastery(gameId, 1) [if unlocks minigame]

GameProgressionService.isMinigameUnlocked(gameId)
  -> checks ALL_STORY_MISSIONS for matching unlocksMinigame + completed

LevelCompletionService.completeLevel(result)
  -> LevelProgressionService.completeLevel(result) [records progress + XP]
  -> MasteryService.updateMastery(gameId) [recalculates stars]
```

## Test Plan

The spec file contains a single `describe` block with 6 test cases:

### Test 1: "completing story mission Ch 1 unlocks Module Assembly minigame"
- Assert `gameProgression.isMinigameUnlocked('module-assembly')` is `false` before.
- Call `completionService.completeMission(1 as ChapterId)`.
- Assert `gameProgression.isMinigameUnlocked('module-assembly')` is `true` after.
- Assert returned summary has `unlockedMinigame === 'module-assembly'`.

### Test 2: "completing story mission unlocks the correct minigame per curriculum mapping"
- Complete missions 1-4 sequentially (to satisfy prerequisite chain).
- Assert `gameProgression.isMinigameUnlocked('flow-commander')` is `true` after Ch 4 (which has `unlocksMinigame: 'flow-commander'`).
- Assert `CurriculumService.getMinigameForChapter(4)` returns `'flow-commander'`.
- Complete missions 5 through 7 sequentially.
- Assert `gameProgression.isMinigameUnlocked('wire-protocol')` is `true` (unlocked by Ch 5).
- Assert `gameProgression.isMinigameUnlocked('signal-corps')` is `true` (unlocked by Ch 7).

### Test 3: "locked minigame cannot be played before its prerequisite story mission"
- Assert `gameProgression.isMinigameUnlocked('flow-commander')` is `false` initially.
- Complete only Ch 1 (which unlocks module-assembly, not flow-commander).
- Assert `gameProgression.isMinigameUnlocked('flow-commander')` is still `false`.
- Assert `gameProgression.isMinigameUnlocked('module-assembly')` is `true`.

### Test 4: "XP is awarded for story mission completion (50 XP per mission)"
- Assert `xpService.totalXp()` is `0` initially.
- Complete Ch 1.
- Assert `xpService.totalXp()` is `50`.
- Complete Ch 2.
- Assert `xpService.totalXp()` is `100`.

### Test 5: "XP is awarded for minigame level completion alongside story XP"
- Complete Ch 1 (awards 50 XP, unlocks module-assembly).
- Register a test level definition for `module-assembly`.
- Call `levelCompletionService.completeLevel()` with a `MinigameResult` for that level (Basic tier, non-perfect).
- Assert `xpService.totalXp()` is `50 + 15 = 65` (50 story + 15 basic tier).

### Test 6: "mastery is set to 1 star on story unlock, updated to calculated value on level completion"
- Assert `masteryService.getMastery('module-assembly')` is `0`.
- Complete Ch 1.
- Assert `masteryService.getMastery('module-assembly')` is `1` (from `ensureMinimumMastery`).
- Register test levels and complete a basic-tier level.
- Assert `masteryService.getMastery('module-assembly')` is `1` (still 1 since only 1 level completed = star 1, and floor is already 1).
