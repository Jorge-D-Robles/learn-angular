# Plan: T-2026-221 — Wire DegradationAlertComponent "Practice Now" to refresher challenge route

## Task

DegradationAlertComponent (T-2026-161) has a "Practice Now" button that emits a `practiceRequested` event with topicId. RefresherChallengePage (T-2026-217) handles the actual practice flow at `/refresher/:topicId`. This ticket wires the button event to router navigation in the DashboardPage and ProfilePage.

## Architecture

### Already Done (by T-2026-233)

After investigating the codebase, **all DashboardPage wiring is already complete**. T-2026-233 ("Wire DegradationAlertComponent into DashboardPage for spaced repetition alerts") was completed on 2026-03-24 and delivered:

1. `DegradationAlertComponent` imported and declared in DashboardPage's `imports` array.
2. `degradingTopicItems` computed signal mapping `SpacedRepetitionService.getDegradingTopics()` to `DegradingTopicItem[]`.
3. Template binding: `(practiceRequested)="navigateToRefresher($event)"` in `dashboard.html` line 74.
4. `navigateToRefresher(topicId: string)` method in `dashboard.ts` line 143-145, calling `this.router.navigate(['/refresher', topicId])`.
5. Test: "should navigate to refresher when practiceRequested fires" in `dashboard.spec.ts` line 314-329, verifying the full click-to-navigate flow.

This satisfies acceptance criteria 1 and most of criteria 4.

### Remaining Work: ProfilePage degradation display

The ticket's acceptance criterion 3 states: "ProfilePage also wires the same navigation for its degradation display." The ProfilePage currently:

- Has a mastery table showing all 12 topics with effective mastery stars.
- Does NOT have a `DegradationAlertComponent` instance.
- Does NOT import `Router`.
- Does NOT have any navigation to `/refresher/:topicId`.

The `navigation.md` design spec for the Profile page does NOT list "degradation alerts" as a UI element. The Profile page's mastery table already shows effective (degraded) mastery stars. Adding a full `DegradationAlertComponent` section would be reasonable for discoverability, but it was not part of the original design spec.

### Decision

Given the ticket's explicit acceptance criterion 3, we will add a `DegradationAlertComponent` to ProfilePage below the mastery table. This requires:

1. Adding `Router` injection to ProfilePage.
2. Adding `DegradationAlertComponent` to imports.
3. Adding `SpacedRepetitionService.getDegradingTopics()` mapped to `DegradingTopicItem[]` (SpacedRepetitionService is already injected).
4. Adding a `navigateToRefresher(topicId: string)` method.
5. Adding the template binding in ProfilePage's template.
6. Two new tests for ProfilePage.

The DashboardPage work is already done and needs zero changes.

### Detailed implementation notes

#### profile.ts changes

**New imports:**
```typescript
import { Router } from '@angular/router';
import type { DegradingTopicItem } from '../../shared/components/degradation-alert/degradation-alert';
import { DegradationAlertComponent } from '../../shared/components';
```

**New injection:**
```typescript
private readonly router = inject(Router);
```

**Add to `imports` array:**
```typescript
DegradationAlertComponent,
```

**New computed signal** (follows same pattern as DashboardPage's `degradingTopicItems`):
```typescript
readonly degradingTopicItems = computed<DegradingTopicItem[]>(() => {
  return this.spacedRepetition.getDegradingTopics().map((t) => ({
    topicId: t.topicId,
    topicName: this.registry.getConfig(t.topicId)?.name ?? t.topicId,
    currentMastery: t.rawMastery,
    effectiveMastery: t.effectiveMastery,
    daysSinceLastPractice: t.daysSinceLastPractice,
  }));
});
```

**New method:**
```typescript
navigateToRefresher(topicId: string): void {
  this.router.navigate(['/refresher', topicId]);
}
```

**Template addition** (after the mastery table section, before the achievements section):
```html
<section class="profile__degradation-section">
  <nx-degradation-alert
    [degradingTopics]="degradingTopicItems()"
    variant="full"
    (practiceRequested)="navigateToRefresher($event)" />
</section>
```

Using `variant="full"` since ProfilePage is a detailed view (vs. "compact" on dashboard).
